# 🧑‍💻 Dev News Daily — April 16, 2026

## 🔥 Top Stories

### 1. Vite 8 + Rolldown: 10-30x Faster Builds — 🔴 Breaking/Major
**[Vite 8 team claims 10-30x faster builds with Rolldown — The Register](https://www.theregister.com/2026/03/16/vite_8_rolldown/)** | **[Vite 8.0 is out! — vite.dev](https://vite.dev/blog/announcing-vite8)**

Lanzado el 12 de marzo y ya en su patch `v8.0.8` (9 de abril), Vite 8 reemplaza completamente esbuild + Rollup por **Rolldown** (Rust) como bundler unificado. Los números son brutales: dev server startup 3x más rápido, full reloads 40% más rápidos, y 10x menos network requests. El stack completo ahora es Vite (build tool) + Rolldown (bundler) + Oxc (compiler), todos coordinados. Si estás en producción con Vite 6 o 7, esto es una migración que vale la pena planificar.

> *Hermano, cuando el equipo de Vite dice "10-30x" y The Register lo verifica, es hora de actualizar el pipeline.*

### 2. TypeScript 6.0 — La última versión en JS antes del salto a Go — 🔴 Breaking/Major
**[Announcing TypeScript 6.0 — devblogs.microsoft.com](https://devblogs.microsoft.com/typescript/announcing-typescript-6-0/)** | **[TypeScript 6.0 RC — devblogs.microsoft.com](https://devblogs.microsoft.com/typescript/announcing-typescript-6-0-rc/)**

TypeScript 6.0 (GA: marzo 17) es el **último release compilado en JavaScript** antes de TypeScript 7 (reescrito en Go). Cambios clave: `strict: true` habilitado por defecto, ES2015 como target mínimo (adiós ES3/ES5), soporte para `Temporal` types, subpath imports con `#/`, y la nueva lib `es2025`. Esto es una release de *transición y limpieza*, no de features. Todo lo deprecado en 6.0 se elimina en 7.0, así que el momento de limpiar la config es ahora.

---

### 3. Next.js 16.2 — Dev Server 400% Más Rápido con Turbopack — 🔴 Breaking/Major
**[Next.js 16.2.3 LTS — releasebot.io](https://releasebot.io/updates/vercel/next-js)** | **[Next.js 16.1 Blog — nextjs.org](https://nextjs.org/blog/next-16-1)**

Next.js 16.2 (marzo 18) + patch 16.2.3 LTS (abril 8) con la integración profunda de Turbopack entregó startup times ~400% más rápidos y rendering ~50% más rápido. También incluye Server Fast Refresh (antes solo en client components), Subresource Integrity support, y file system caching mejorado. El patch 16.2.3 además parchea dos CVEs (`CVE-2026-27979` y `CVE-2026-29057`). Actualización de seguridad obligatoria.

**[TypeScript 6 Beta — InfoQ](https://www.infoq.com/news/2026/02/typescript-6-released-beta/)** 🟡 Notable
TypeScript 6.0 como release de transición hacia el Go rewrite. Novedades: `ignoreDeprecations: "6.0"` para suavizar la migración, SimpleChanges ahora es genérico, y target ES3/ES5 deprecados. El path hacia TypeScript 7 (nativo, 8x más rápido en project load) está trazado.

El motor nativo (Go) de TypeScript 7 está en preview en VS 2026 Insiders. El language service muestra ~8x de mejora en project load times. No hay fecha oficial de GA todavía.

### 🎨 CSS

**[CSS Snapshot 2026 — W3C](https://www.w3.org/TR/css-2026/)** 🟡 Notable
El CSS Working Group publicó el snapshot 2026 consolidando el estado actual de specs. Highlights que llegan a baseline este año: **CSS Mixins nativos** (`@mixin`/`@apply`), **`@starting-style`** cross-browser (entry animations sin JS), y **View Transitions para MPAs**. `@scope` ya es production-ready en Chrome y Safari.

**[Interop 2026 — Exciting New CSS Features — medianic.co.uk](https://www.medianic.co.uk/2026/03/04/interop-2026-exciting-new-css-features-and-browser-support/)** 🟢 Minor
Firefox 147 habilitó CSS Anchor Positioning y la Navigation API, que ahora alcanzan Baseline Newly Available. `sibling-index()` / `sibling-count()` aún en desarrollo, se espera baseline mid-late 2026.

**[Tailwind CSS v4 — tailwindcss.com](https://tailwindcss.com/blog/tailwindcss-v4)** 🟡 Notable
Tailwind v4.1 (abril 2026) con mejoras sobre el v4.0 que reescribió el engine en Rust. Builds full 5x más rápidos, incremental en microsegundos, zero-config automático y sin `tailwind.config.js`. Requiere Safari 16.4+, Chrome 111+, Firefox 128+.

### 🌐 HTML / Web Platform

**[Chrome 144: `<geolocation>` element y `::search-text` — web.dev](https://web.dev/blog/web-platform-01-2026)** 🟡 Notable
Chrome 144 introduce el elemento declarativo `<geolocation>` que simplifica el flujo de permisos de ubicación, y el pseudo-elemento `::search-text` para estilizar resultados de find-in-page (similar a `::selection`). Ambos llegan a Baseline 2026.

**[Navigation API — Baseline Newly Available — web.dev](https://web.dev/baseline)** 🟢 Minor
Con Firefox 147, la Navigation API alcanza Baseline Newly Available. Permite interceptar navegaciones del cliente sin depender del History API heredado — fundamental para routers de SPA.

### 📱 React Native

**[Expo SDK 55 — expo.dev/changelog/sdk-55](https://expo.dev/changelog/sdk-55)** 🔴 Breaking/Major
SDK 55 (GA: febrero 25) **elimina definitivamente la Legacy Architecture**. Solo New Architecture de aquí en adelante.
