<div class="update-content">
<h3>Axios supply chain compromiso — CVE-2026-40175</h3>
<p>Ojo con este, actualizá YA. Una cuenta de maintainer comprometida publicó versiones maliciosas de axios (1.14.1 <code>latest</code> y 0.30.4 <code>legacy</code>) con un Remote Access Trojan multiplataforma vía <code>postinstall</code>. Atribuido el 1 de abril a <strong>UNC1069, un actor norcoreano</strong>. La versión limpia es <strong><code>axios@1.15.0</code></strong> (publicada el 7 de abril). Si estás corriendo CI en un runner con conexión a internet y no actualizaste, tenés un problema serio. ~100M descargas semanales. Actualizar a 1.15.0 inmediatamente.</p>
<a href="https://www.elastic.co/security-labs/axios-one-rat-to-rule-them-all">Elastic Security Labs</a>
</div>
</div>
<div class="update-item">
<span class="update-tag tag-breaking">Breaking</span>
<div class="update-content">
<h3>Node.js — CVEs críticos parcheados (13 abril)</h3>
<p>Múltiples CVEs en Node.js 20.x y 24.x: DoS por header <code>__proto__</code> manipulado (CVE-2026-21710), V8 string hashing (CVE-2026-21717), nghttp2 DoS (CVE-2026-27135), y varios en undici (WebSocket/DoS). El <code>__proto__</code> header injection es una superficie de ataque real en cualquier servidor HTTP que procese input no confiable. Actualizá Node LTS ahora si no lo hiciste la semana pasada.</p>
<a href="https://nodejs.org/en/blog/vulnerability">nodejs.org</a>
</div>
</div>

<span class="update-tag tag-notable">Notable</span>

<h3>AWS Interconnect Multicloud — GA el 15 de abril</h3>
<p>Conexión privada directa entre tu VPC en AWS y VPCs en otros cloud providers, sin pasar por internet público. Lanzó primero con Google Cloud en 5 pares de regiones (N. Virginia, Oregon, Londres, Frankfurt + California↔LA). Azure y OCI llegan después en 2026. Tráfico viaja 100% por el backbone de AWS + red privada del partner. Eso es predicibilidad de latencia real.</p>
<a href="https://aws.amazon.com/blogs/aws/aws-interconnect-is-now-generally-available-with-a-new-option-to-simplify-last-mile-connectivity/">AWS Blog</a>
</div>
</div>
<div class="update-item">
<span class="update-tag tag-notable">Notable</span>
<div class="update-content">
<h3>Cloudflare Dynamic Workers — Open Beta (13 abril)</h3>
<p>Permiten que un Worker instancie otro Worker en runtime con código generado dinámicamente, cada uno en su propio isolate V8. <strong>100x más rápido que containers</strong> (arranque en milisegundos, no segundos) y 10-100x más eficiente en memoria. El caso de uso target: ejecutar código generado por AI agents de forma segura. Ya disponible en open beta para paid Workers.</p>
<a href="https://blog.cloudflare.com/dynamic-workers/">Cloudflare Blog</a>
</div>
</div>
<div class="update-item">
<span class="update-tag tag-notable">Notable</span>
<div class="update-content">
<h3>TypeScript 7 — Preview nativo en VS 2026 (Go rewrite, 10x faster)</h3>
<p>TS 7 es la reescritura en Go. En preview activo ahora dentro de VS 2026 Insiders. Breaking changes importantes: <strong>strict-by-default, target ES5 eliminado, AMD/UMD/SystemJS removidos</strong>. TS 6.0 (marzo 23) fue el puente: es la última versión basada en JS, con nuevos defaults y deprecaciones que preparan el terreno. Revisá tu <code>tsconfig.json</code> antes de saltar.</p>
<a href="https://developer.microsoft.com/blog/typescript-7-native-preview-in-visual-studio-2026">Microsoft Developer Blog</a>

<h3>OpenTelemetry — Declarative Config estable (abril)</h3>
<p>La spec de configuración declarativa de OTel alcanzó status stable. Un solo archivo YAML define trace, metric y log pipelines — lo apuntás con <code>OTEL_CONFIG_FILE</code> y listo. Históricamente esto era un desastre de env vars + código SDK + configs del collector por separado. El cuarto signal — <strong>continuous profiling</strong> — entró en release candidate en Q1 2026. OTel es el primer estándar open que unifica los 4 pillars bajo un SDK, un wire protocol (OTLP).</p>
<a href="https://www.infoq.com/news/2026/04/opentelemetry-declarative-config/">InfoQ</a>
</div>
</div>
