## What it is

Four autonomous Claude Code scheduled tasks generate daily briefs and push them to this repo. A GitHub Action deploys the blog to GitHub Pages on every commit. Another cleanup workflow prunes briefs older than 30 days.

### Categories

| Category | Schedule (local, Mon–Fri) | What it covers |
| -------- | ------------------------- | -------------- |
| 🤖 **AI Coding** | 09:09 | Claude Code, Codex, Copilot, Gemini + model launches (OpenAI, Anthropic, Qwen, DeepSeek, Kimi, GLM, Doubao, MiniMax) |
| 🧑‍💻 **Dev News** | 08:24 | TypeScript, CSS, HTML, React, React Native, frameworks, web platform |
| ⚙️ **Backend & Fullstack** | 08:54 | TypeScript, Node.js, Deno, Bun, databases, DevOps, architecture |
| 🚀 **Gentleman Releases** | 09:34 | Release digest for Gentleman-Programming OSS (engram, gentle-ai) |

All briefs are trilingual (ES / EN / FR) with a language toggle.

## How it works

```
┌────────────────────────────────────────────────────────────────┐
│  Claude Code scheduled tasks (run on local machine)            │
│     └── generate brief (.json) → git commit → git push         │
│                                        ↓                        │
│  GitHub Actions: Deploy to GitHub Pages (on push)              │
│     └── pnpm install → astro build → deploy                    │
│                                        ↓                        │
│  Live at https://ovmelo03031.github.io/dev-daily-briefs/       │
│                                                                │
│  GitHub Actions: Cleanup (daily at 15:17 UTC)                  │
│     └── delete briefs older than 30 days → push                │
└────────────────────────────────────────────────────────────────┘
```

## Stack

- **[Astro](https://astro.build)** 6.1 — static site generator with content collections
- **[Tailwind CSS](https://tailwindcss.com)** 4 — styling
- **JSON content + Astro components** — each brief is structured `.json` (trilingual ES/EN/FR) rendered by typed Astro components
- **[astro-icon](https://www.astroicon.dev/)** + Iconify (`simple-icons`, `lucide`) — brand/UI icons
- **pnpm** — package manager (with `ignore-scripts=true` for security)
- **GitHub Pages + Actions** — hosting and CI/CD (cost: $0 for public repos)

## Local development

```sh
pnpm install    # also wires the pre-push hook (core.hooksPath → .githooks)
pnpm dev        # http://localhost:4321
pnpm build      # production build
```

> **Pre-push hook:** `pnpm install` runs a `prepare` script that points git at
> `.githooks/`. The `pre-push` hook runs a full `pnpm build` before every push,
> so a broken build (e.g. an invalid icon name in a brief) fails locally instead
> of in GitHub Actions. Bypass with `git push --no-verify` when needed.

### Commands

| Command | What it does |
| ------- | ------------ |
| `pnpm dev` | Start dev server |
| `pnpm build` | Build for production |
| `pnpm verify` | Build to validate before pushing (same as the pre-push hook) |
| `pnpm preview` | Preview the production build locally |
| `./scripts/cleanup-old-briefs.sh --dry-run` | See which briefs would be cleaned up |
| `node scripts/generate-brand-assets.mjs` | Regenerate favicons and OG image from source logo |
| `gh workflow run cleanup.yml` | Manually trigger the cleanup workflow |

## Retention policy

Only the last **30 days** of briefs are kept on the site. Older briefs are removed daily at 12:17 PM (UTC-3) by a dedicated cleanup workflow. Git history retains all historical briefs — nothing is lost forever.
