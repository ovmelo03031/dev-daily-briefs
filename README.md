# Dev Daily Briefs

Automated daily digest of developer news, delivered every morning.

**🌐 Live site:** [ovmelo03031.github.io/dev-daily-briefs](https://ovmelo03031.github.io/dev-daily-briefs/)

</div>

---

## What it is

Three autonomous Claude Code scheduled tasks generate daily briefs and push them to this repo. A GitHub Action deploys the blog to GitHub Pages on every commit. Another cleanup workflow prunes briefs older than 30 days.

### Categories

| Category | Schedule (local) | What it covers |
| -------- | ---------------- | -------------- |
| 🤖 **AI Coding** | 10:30 daily | Claude Code, Codex, Copilot, Gemini + model launches (OpenAI, Anthropic, Qwen, DeepSeek, Kimi, GLM, Doubao) |
| ⚙️ **Backend & Fullstack** | 08:45 daily | TypeScript, Node.js, Deno, Bun, databases, DevOps, architecture |
| 🧑‍💻 **Dev News** | 08:15 daily | TypeScript, CSS, HTML, React, React Native, frameworks, web platform |

All briefs are bilingual (EN / ES) with a language toggle.

## How it works

```
┌────────────────────────────────────────────────────────────────┐
│  Claude Code scheduled tasks (run on local machine)            │
│     └── generate brief (.mdx) → git commit → git push          │
│                                        ↓                        │
│  GitHub Actions: Deploy to GitHub Pages (on push)              │
│     └── pnpm install → astro build → deploy                    │
│                                        ↓                        │
│  Live at https://ovmelo03031.github.io/dev-daily-briefs/       │
│                                                                │
│  GitHub Actions: Cleanup (daily at 15:17 UTC)                  │
│     └── delete briefs older than 30 days → push                │
└────────────────────────────────────────────────────────────────┘
```

## Stack

- **[Astro](https://astro.build)** 6.1 — static site generator with content collections
- **[Tailwind CSS](https://tailwindcss.com)** 4 — styling
- **[MDX](https://mdxjs.com/)** — bilingual markdown with embedded HTML cards
- **pnpm** — package manager (with `ignore-scripts=true` for security)
- **GitHub Pages + Actions** — hosting and CI/CD (cost: $0 for public repos)

## Local development

```sh
pnpm install
pnpm dev        # http://localhost:4321
pnpm build      # production build
```

### Commands

| Command | What it does |
| ------- | ------------ |
| `pnpm dev` | Start dev server |
| `pnpm build` | Build for production |
| `pnpm preview` | Preview the production build locally |
| `./scripts/cleanup-old-briefs.sh --dry-run` | See which briefs would be cleaned up |
| `node scripts/generate-brand-assets.mjs` | Regenerate favicons and OG image from source logo |
| `gh workflow run cleanup.yml` | Manually trigger the cleanup workflow |

## Retention policy

Only the last **30 days** of briefs are kept on the site. Older briefs are removed daily at 12:17 PM (UTC-3) by a dedicated cleanup workflow. Git history retains all historical briefs — nothing is lost forever.
